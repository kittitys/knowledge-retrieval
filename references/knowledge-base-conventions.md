# 知识库结构与目录规范

> 本文档覆盖 knowledge-base 目录结构、data_structure.md 模板、快捷方式通路设计。
> 在 Stage 0 初始化新项目、或 Phase 0 检查索引新鲜度时按需查阅。

---

## 1. 核心设计原则

**原始文件保留在原始位置（OneDrive / 本地文件夹），不搬入 workspace。**

`knowledge-base/` 只存放：

| 内容 | 用途 | 管理方式 |
|------|------|---------|
| `data_structure.md` | 文件级索引（文件名、描述、位置路径） | AI 自动维护 |
| `cache/` | 按需生成的提取缓存 | 自动管理 |
| `.bm25_index/` | BM25 全文搜索索引 | 自动管理 |
| `快捷方式` | 原始文件夹 ↔ workspace 的双向通路 | 一次性创建 |

**索引粒度：文件级，非目录级。** 子目录是文件的一个属性字段（分类标签），不是搜索入口。

---

## 2. 目录规范

```
knowledge-base/<项目名>/
├── data_structure.md           ← 文件级索引
├── .bm25_index/                ← BM25 索引（自动管理）
│   └── index/
└── cache/                      ← 按需生成的缓存（自动管理）
    ├── <文件名>.txt            ← PDF/PPTX 提取文本缓存
    └── <文件名>.img-<页码>.txt ← 图片解读缓存
```

---

## 3. data_structure.md 模板

每个项目一个。核心是文件级索引表。

```markdown
# <项目名> — 原始素材索引

## 实际位置
> <原始文件夹绝对路径>

## 文件索引

| 文件名 | 类型 | 描述 | 位置 |
|--------|------|------|------|
| 项目立项流程.pdf | 制度文件 | 公司内部项目立项流程与审批规范 | 政策文件/ |
| 行业政策汇编（2024）.pdf | 政策文件 | 国家发改委年度行业政策汇编 | 行业相关/ |
| …更多条目同上格式 | | | |
```

**说明：**
- `描述`列：AI 初次生成初始描述（基于文件名+目录）+ 每次 Phase B 读完后自动更新
- `位置`列：仅用于拼合完整路径以访问文件，不用于搜索判断
- 不设独立标签列。文件的浓缩信息通过描述列自然进化

## 4. 双向快捷方式通路

### 原始文件夹 → workspace
在原始文件夹根目录创建 `.shortcut.lnk`，指向 `workspace/knowledge-base/<项目名>/`。

### workspace → 原始文件夹
在 `workspace/knowledge-base/<项目名>/` 创建 `.shortcut.lnk`，指向原始文件夹路径。

### 完整通路
```
原始文件夹（用户日常在此）          knowledge-base/<项目>/
        │                                    │
        ├── .shortcut.lnk ───────────────────┤
        │    → knowledge-base/<项目>/         │
        │                                    ├── .shortcut.lnk
        │                                    │    → 原始文件夹路径
        │                                    │
        └── 看 KM 索引 ←→ 看原始文件          ┘
```

## 5. Stage 0 — 知识库初始化（一次性注册）

### 输入
用户提供原始文件夹的绝对路径（如 `~/OneDrive/工作/XX项目`）。

### 执行流程

**Step 1 — 创建工作区副本：**
- 在 `workspace/knowledge-base/` 下创建目录，目录名 = 原始文件夹名

**Step 2 — 扫描并生成文件索引：**
- 遍历原始文件夹下所有文件
- 自动推断：文件名、类型、初始描述、位置
- 写入 data_structure.md

**Step 3 — 构建 BM25 搜索索引：**
```bash
python .agents/skills/knowledge-retrieval/scripts/build_kb_index.py --project <文件夹名>
```

**Step 4 — 创建双向快捷方式：**
- 原始文件夹侧 → 指向 workspace 副本
- workspace 侧 → 指向原始文件夹路径

**Step 5 — 告知用户：**
- 知识库已就绪，可以开始搜索
- 扼要报告：文件数、索引大小

### 分阶段汇报
注册过程 5-15 秒，分阶段向用户汇报：
> ① 正在扫描文件列表…（120 个文件）
> ② 正在构建搜索索引…（约 5 秒）
> ③ ✅ 知识库已就绪。120 个文件，可以开始搜索。

### 再次执行 Stage 0 的条件
- 注册新项目 → 正常执行一次
- 原始文件夹搬迁 → 走 Phase 0 自动处理，不需要重新 Stage 0
- 日常文件增删 → 走 Phase 0 索引新鲜度检查自动处理
