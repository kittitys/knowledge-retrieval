## 优先级排序（降序）

### 1. 溯源锚点 — 回答末尾附带深度链接

**现状：** Phase B 会读取文件但不在回答中附带来源定位
**目标：** 每次检索回答末尾强制附带 `[来源: 文件名#段落]` 格式的链接
**投入：** 低 — 在 SKILL 输出规范中加一条硬约束即可

### 2. 交叉比对 / 冲突检测模式

**现状：** 多文件阅读后只做汇总，不主动比较不同文件间的矛盾
**目标：** 当 Phase B 读取 ≥ 2 个文件且发现观点不一致时，自动生成「矛盾提示卡」
**投入：** 中 — 需要新增 Phase B 子模式 + Mermaid 表格输出能力

### 3. MECE 结构化输出

**现状：** 回答偏向自然语言描述
**目标：** 在检索类问题上强制使用表格、金字塔结构、对比矩阵等 MECE 格式输出
**投入：** 低 — 在 SKILL 的输出规范中增加格式偏好指令

### 4. 一键安装脚本（待可行性研究）

**来源：** 豆包评测建议
**问题：** 非技术用户需手动 pip install，门槛偏高
**方向：** 提供 `scripts/setup.bat` 一键安装依赖。需研究：Python 环境检测、依赖冲突处理、无管理员权限时的降级方案
**优先级：** 高 — 直接影响用户首次体验
**状态：** 待研究 ✅ 已记录

### 5. 索引备份与恢复（待可行性研究）

**来源：** 豆包评测建议 + 我们自己的删库事件教训
**问题：** BM25 索引损坏时自动降级到语义搜索，但没有恢复机制
**方向：** 自动定期备份 `.corpus/` 到 `.corpus-backup/`，增量备份 + 一键恢复
**优先级：** 中 — 不常见但遇到就很痛
**状态：** 待研究 ✅ 已记录

### 6. 向量搜索（无限期搁置）

**原因：** 用户反馈 → 溯源 + 冲突检测 + MECE 输出 比 BM25 的命中率提升更重要
**恢复条件：** 前三项完成后发现语义匹配仍有不足时重新评估
